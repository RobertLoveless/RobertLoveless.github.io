FLUTTER ENVIROMENT SETUP For Ubuntu 18.04

Step 1: Install cURL & JDK

Give current user write permission
sudo chown -R $USER:$USER /opt
Install cURL (client URL) command line tool for file transfer with a URL syntax.
It supports several protocols including HTTP, HTTPS, FTP, and many more.
sudo apt-get install curl
First well need the Java Development Kit(JDK)
JDK can be installed on our Ubuntu system with the below commands.
sudo add-apt-repository ppa:openjdk-r/ppa
sudo apt-get update
sudo apt-get install openjdk-8-jdk

Step 2: Install and Set Up Android Studio

Android studio needs to be installed for working with Flutter development.
Below is the official android Studio zip file link.
https://developer.android.com/studio/index.html
Unzip the file to /opt directory by running the command.
sudo unzip android-studio-ide-*version* -d /opt
Configure ANDROID_HOME Environment variable
Environment variables are global system variables accessible by all the processes running under our Operating System.
So open .bash_profile file using nano or vim editor.
Extract the file to /opt
copy and paste the below code in it.
export ANDROID_HOME=$HOME/Android/sdk
export PATH=$PATH:$ANDROID_HOME/tools
export PATH=$PATH:$ANDROID_HOME/platform-tools
Install Flutter Plugin on Android Studio
Go to, Configure -> Plugin
Now, Search of Flutter and Install. It requires a restart for Android Studio to activate this plugin.

Step 3. Install and Set Up Flutter SDK

To start creating Flutter projects, Flutter Software Development Kit (SDK) is necessary. This can be installed on our system with the below steps.
https://flutter.dev/docs/get-started/install/linux
Install & Download the tar.xz file and Extract the File to /opt
tar -xf file.name.tar -C /path/to/directory
Now we need to add the Flutter to our systemâ€™s path using the below command.
export PATH=`pwd`/flutter/bin:$PATH
source $HOME/.bash_profile

Step 4. Accept the Licenses

We need to accept some licenses to start development with Flutter. Just run the below command and accept all the licenses by pressing y.
flutter doctor --android-licenses

Step 5. Update the Path Permanently

Edit .bashrc and add this path to make the it a permanent change
sudo nano .bashrc
export PATH=/opt/android-studio/bin:/opt/flutter/bin:$PATH

Step 6: Choosing the Code Editor

You can now use either android studio to starting creating flutter apps or you can download visual studio code which is also directly supports the Flutter and Flutter plugin.
Link below
https://code.visualstudio.com/#alt-downloads
For Android
On our mobile device,
Go to Settings->About
We can see a Build Version menu there. Continuously click the option until it enables the Developer options
Now, return to Settings. We can see the Developer options menu there. Enable it. Also, enable the USB debugging option in it.
Connect our mobile device with our Ubuntu system using the USB port. Type the below command to check whether our mobile device is connected or not.
flutter devices
If it is connected, from the VS studio terminal, run
flutter run
This will install and open the newly created app on our connected Android device.
For Web
In order to enable for web, run these commands:
flutter channel beta
flutter upgrade
flutter config --enable-web

This command displays any issues with flutter and the steps to take to remedy the issue.
flutter doctor

LXC Commands - Breaking Bridge and SSH

LXC is an operating-system-level virtualization method for running multiple isolated Linux systems on a control host using a single Linux kernel.

Step 1: Turn off containers

sudo lxc stop "container"

Step 2: Remove eth0 device

sudo lxc config device remove "container name" eth0

Step 3: Follow this.

https://discuss.linuxcontainers.org/t/forward-port-80-and-443-from-wan-to-container/2042
use the add command listed:
lxc config device add "container name" "proxyname ("anyname")" proxy listen=tcp:0.0.0.0:"Any port" connect=tcp:127.0.0.1:22 (22 is the default ssh port)

Step 4: Turn on your container and enter with below command

sudo lxc exec "container name" bash

Step 5: Modify "/etc/ssh/sshd_config"

Change: PermitRootLogin yes
Change: PasswordAuthentication yes

Step 6: Give your root user a password

sudo passwd root

Step 7: Restart your sshd service

sudo systemctl restart sshd

Step 8: To test ssh in, open powershell on your PC and enter:

ssh root@10.100.3.43 -p "whatever port you set"

sudo lxc config device add W0416792-lamp proxy listen=tcp:0.0.0.0:"Any port" connect=tcp:127.0.0.1:80

Basic LXC commands
View all running containers
COMMAND: lxc list
Set a Container to use a bridge connection
COMMAND: $ lxc config device add containername eth0 nic nictype=bridged parent=br0 name=eth0
Stop a container
COMMAND: lxc stop
Start a container
COMMAND: lxc start
Access the terminal shell on the running container
COMMAND: lxc exec /bin/bash
Send a command directly to the container from the hosts terminal
COMMAND: lxc exec apt-get update
Send a file to the container
COMMAND: lxc file push //
Pull a file from the container
COMMAND: lxc file pull first/etc/hosts .
